5. コンテナ基盤（ECS / Fargate）
5-1. ECSの基本構成

Task：コンテナ実行単位

Service：Taskを管理（AutoHeal / LB連携）

Cluster：リソースの集合（Fargateは管理不要）

5-2. タスク定義

CPU / Memory は明確に指定

Secrets Manager / SSM Parameter Store

イメージはECR推奨

5-3. ECSサービス

デプロイ方式：Rolling / Blue-Green

AutoHeal：異常タスクの再起動

5-4. Fargate vs EC2起動

Fargate：サーバレス。管理不要

EC2：柔軟だがメンテ必要

5-5. ECS + ALB

ターゲットグループにDynamic Port Mapping

ECS1サービス＝1TGが基本

5-6. Auto Scaling

CPU / Memory / ALB RequestCount

スケジュール（夜間停止/縮小など）

5-7. ECR設計

リポジトリ単位はサービス毎

ライフサイクルで古いイメージ削除

5-8. CloudWatch Logs

すべてのタスクはログを1箇所に集める

フィルタで監視可
